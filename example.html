02
Stateless JWT access tokens (15m), rotating refresh tokens (7d) stored in Redis.
approach · changed today
WHY15m access tokens limit blast radius of token theft. Refresh rotation enables server-side revocation while keeping the access path stateless. Redis chosen for sub-5ms reads (Postgres would be 15-30ms p99 at our row count).
DECISION docs/adr/015-jwt-refresh-rotation.md
BENCHMARK scripts/bench/token-lookup.py · 2026-05-24 · Redis p99=2.1ms vs Postgres p99=24ms
ALTERNATIVES CONSIDERED
REJECTED Session cookies (stateful, doesn't scale to multi-region)
REJECTED Long-lived JWTs without refresh (no revocation path)
REJECTED OAuth pass-through to upstream IdP (adds external dependency, costs)
